Creating Accountability Plan

  • 0:00 - 3:00 Creating individual steps of the Accountability Plan (Implementation Stage) for capturing information about the produced ML model. The steps cover the same information that is captured by the Model Card Toolkit.
  • 3:00 - 8:12 Enriching the plan description with describtion of input and output variables for each individual step. Human validated constrain is defined (6:50 - 7:20). A user completing accountability trace form will have to manually answer whether this constraint has been satisfied.
  • 8:12 - 8:50 Constructing SHACL constraint using the build in template to require information element rains:Limitation to be provided in the part of the accountability trace describing the implemented model.
  • 8:50 - 10:09 The plan description is completed after required input and out variables are defined for the model evaluation.

Model Card Data Converter

Thsi video demostrates the usage of a Rains Model Card Data Converter which produces accountability trace from data captured by Google's Model Card Toolkit. The example is based on the official Model Card toolkit example

  • 0:00 - 1:56 An overview of the code for training Keras DNN classifier (part of the original example)
  • 1:56 - 2:46 An overview of the code for producing Model Card (part of the original example)
  • 2:46 - 3:53 An overview of the code for producing accountability trace from Model Cards with a failed constraint check (Rains contribution)
  • 3:53 - 5:18 A process of adding the information about license to the model card and re-creating the accountability trace (Rains contribution)
